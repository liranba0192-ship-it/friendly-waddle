# שיעור פיננסי שבועי חדש 💰📈

> זהו ה-prompt שטריגר מתוזמן (Scheduled Trigger) של Claude Code מריץ **פעם בשבוע**.
> המטרה: להוסיף **שיעור פיננסי חדש אחד** למסלול הלימוד באפליקציה, כדי שהמסלול ימשיך לגדול ולא יחזור על עצמו.
> חבר אליו טריגר שבועי (ראה `SETUP.md`).

## המשימה
הוסף **שיעור פיננסי אחד חדש** לקובץ `app/data/finance.json` — קצר, ברור, בעברית, מבוסס ומעשי.

## 🔄 נושא חדש (חובה — קרא קודם!)
1. קרא את `app/data/finance.json` ועבור על **כל השיעורים הקיימים** (שדות `id` ו-`title`).
2. בחר **נושא חדש שלא קיים עדיין**. הנושאים הבסיסיים כבר כוסו (כסף, תקציב, חיסכון, ריבית דריבית, חובות, אשראי, השקעות, סיכון, מסים, פנסיה, ביטוח, הונאות).
   העדף נושאים **מתקדמים/מעשיים** הבאים בתור, למשל:
   - משכנתא — איך היא עובדת ואיך בוחרים מסלול
   - קרן השתלמות לעומק / קופות גמל
   - מדד המחירים והצמדה
   - תקציב לזוג / משק בית
   - קנייה מול שכירות של דירה
   - מטבעות דיגיטליים — מה זה ולמה להיזהר
   - דירוג אשראי ונתוני אשראי
   - חיסכון לילדים
   - עצמאי מול שכיר — היבטים כלכליים
   - תכנון פיננסי לטווח ארוך / עצמאות כלכלית (FIRE)
   - פסיכולוגיה של כסף וטעויות נפוצות
   - אינפלציה ודפלציה לעומק
3. אפשר WebSearch עם השנה הנוכחית כדי לוודא נתונים עדכניים (אחוזי מס, סכומים בישראל וכו').

## כללי איכות (קריטי)
1. **אל תמציא מספרים/אחוזים/סכומים.** כל נתון ישראלי (אחוז מס, תקרה, סכום) חייב להיות נכון ועדכני — ודא ב-WebSearch. אם לא בטוח — נסח כללי בלי מספר מומצא.
2. עברית פשוטה וברורה, מהיסוד, כאילו הקורא לא יודע כלום בנושא. הסבר כל מונח.
3. כלול **דוגמה מספרית** אחת לפחות, וטבלה אם מתאים.
4. אורך: כמו השיעורים הקיימים (~250–450 מילים ב-Markdown).
5. סיים בטיפ אחד מעשי.

## פורמט — הוסף אובייקט אחד למערך `lessons`
הוסף **בסוף** מערך `lessons` (כדי שיופיע אחרי הבסיסיים) אובייקט במבנה הבא:
```json
{
  "id": "<מזהה-באנגלית-קצר-וייחודי>",
  "icon": "<אמוג'י מתאים>",
  "title": "<כותרת קצרה בעברית>",
  "md": "## <כותרת>\\n<תוכן Markdown עם \\n לשורות חדשות, טבלאות, ו-> לציטוט הטיפ>",
  "tip": "<משפט טיפ אחד מעשי>"
}
```
- שדה `md` הוא מחרוזת אחת — השתמש ב-`\\n` למעברי שורה (כמו בשיעורים הקיימים).
- ודא **id ייחודי** שלא קיים כבר.
- ודא שה-JSON **תקין לחלוטין** אחרי ההוספה (פסיקים נכונים, מרכאות תקינות). בדוק עם כלי אם אפשר.

## 💾 שמירה (חובה — בסוף ההרצה)
1. כתוב את `app/data/finance.json` המעודכן (כל השיעורים הקיימים + החדש).
2. ודא תקינות JSON (לדוגמה `python3 -c "import json;json.load(open('app/data/finance.json'))"`).
3. בצע **commit ו-push** ל-branch ברירת המחדל (`main`). הודעת commit: `Add finance lesson: <title>`.

> כך כל שבוע נוסף שיעור פיננסי חדש, והאפליקציה (טאב לימוד → פיננסים) מציגה אותו אוטומטית — המסלול גדל בלי לחזור על עצמו.
