# ידע כללי יומי — היסטוריה, מדע, גיאוגרפיה ותרבות 🧠🌍

> זהו ה-prompt שטריגר מתוזמן (Scheduled Trigger) של Claude Code מריץ כל בוקר.
> המטרה: **נושא ידע כללי אחד קצר ומעניין** — שמופיע בטאב "לימוד → 🧠 ידע כללי" באפליקציה.
> אין צורך לערוך אותו ידנית — פשוט חבר אליו את הטריגר (ראה `SETUP.md`).

## המשימה

הפק **נושא ידע כללי אחד בעברית** — היסטוריה, מדע, גיאוגרפיה, תרבות, אמנות, טבע, טכנולוגיה, פילוסופיה, שפה, חלל וכו'. קצר, מעניין, מדויק עובדתית.
היקף: קריאה של **2–3 דקות** (בערך 250–450 מילים). מוסבר בבהירות, עם עובדות מרתקות ("האם ידעת?").

🎯 **עדיפות קבועה**: ברוב הימים (בערך 2 מתוך כל 3) בחר נושא מתוך שלושת התחומים המועדפים: **חלל ואסטרונומיה**, **גיאוגרפיה** (מדינות, עמים, שפות, תרבויות, אתרים), או **רכב ותחבורה**. ביום השלישי אפשר לגוון לכל תחום ידע כללי אחר מהרשימה למטה. עדיין חובה לבדוק את הפנקס (`general-knowledge-topics.json`) ולא לחזור על נושא — גם בתוך שלושת התחומים המועדפים יש מגוון עצום (כוכבי לכת שונים, מדינות שונות, תקופות רכב שונות וכו').

## 🔄 אסור לחזור על נושא — פנקס חובה (קרא ובצע קודם!)

יש בריפו קובץ **`app/data/general-knowledge-topics.json`** — כל הנושאים שכבר נכתבו אי פעם.

1. **קרא את הקובץ** במלואו. כל מה שמופיע בו — אסור.
2. בחר **נושא חדש לגמרי** שאינו מופיע בו — לא הנושא עצמו ולא וריאציה קרובה.
3. הנושא יכול להיות מכל תחום ידע כללי (לא כושר/תזונה/AI/פיננסים — יש להם כבר מדריכים נפרדים).
4. בסס על **מחקר עדכני ומדויק**: WebSearch לאימות עובדות, תאריכים ומספרים.

## 🗂️ רעיונות לנושאים (בחר רק כאלה שלא בפנקס — אפשר גם מחוץ לרשימה)

**היסטוריה:** אימפריות עתיקות · מהפכות · תגליות גיאוגרפיות · מלחמות משמעותיות · תקופות זהב תרבותיות · אירועים ששינו את העולם

**מדע וטבע:** תגליות פיזיקה/כימיה/ביולוגיה · בעלי חיים מדהימים · תופעות טבע · גוף האדם · אבולוציה · מוח ותודעה

**חלל ואסטרונומיה:** כוכבי לכת · חורים שחורים · משימות חלל · גלקסיות · תיאוריות יקום · משימות חלל עתידיות/פרטיות (SpaceX וכו') · טלסקופים

**גיאוגרפיה ותרבות:** מדינות ועמים · שפות נכחדות/נדירות · מנהגים ייחודיים · אתרי מורשת עולמית · ערים עתיקות · תופעות גיאוגרפיות (פיורדים, מדבריות, אגמים) · גבולות ומדינות ייחודיות

**רכב ותחבורה:** היסטוריית הרכב · מכוניות חשמליות וסוללות · מותגי רכב איקוניים ומקורם · מנועי בעירה מול חשמל מול היברידי · טכנולוגיות בטיחות ברכב · רכב אוטונומי · תחבורה ציבורית מסביב לעולם · רכבות מהירות · מירוצי מכוניות כתופעה טכנולוגית/היסטורית (לא תוצאות/חדשות ספורט עדכניות)

**אמנות ואדריכלות:** תנועות אמנותיות · אדריכלים/אמנים משפיעים · פלאי אדריכלות · מוזיקה קלאסית

**טכנולוגיה והמצאות:** המצאות ששינו את העולם · היסטוריית המחשוב · תגליות רפואיות

**פילוסופיה ושפה:** הוגי דעות · מקורות מילים ושפות · פרדוקסים לוגיים

> 📌 אפשר נושאים שאינם ברשימה, כל עוד הם ידע כללי מבוסס-עובדות ו**לא בפנקס**.

## ⛔ כללי-ברזל נגד מידע לא מבוסס
1. **אסור להמציא** עובדות, תאריכים, שמות או מספרים. אם לא מצאת מקור אמין — אל תכתוב את הטענה.
2. כל תאריך/מספר/שם חייב לבוא ממקור שמצאת ב-WebSearch, לא מהזיכרון בלבד — אמת עובדות שנויות במחלוקת.
3. כשיש אי-ודאות היסטורית או מדעית — אמור זאת במפורש ("יש חילוקי דעות", "לפי המקורות הזמינים").

## 🗣️ סגנון — מעניין ונגיש לכולם
1. הנח שלקורא אין ידע מוקדם בנושא. הסבר כל מושג מאפס.
2. ספר את זה כמו **סיפור מרתק**, לא כרשימת עובדות יבשה.
3. שפה ברורה וחיה, עם דוגמאות והקבלות מהחיים.
4. סיים ב"האם ידעת?" — עובדה מפתיעה אחת שקשורה לנושא.

## 💾 שמירה לאפליקציה (חובה — בסוף ההרצה)

1. **קרא את `app/data/general-knowledge.json`.** מבנה: `{ "note": ..., "lessons": [ ... ] }`.
2. **הוסף את הנושא של היום בסוף מערך `lessons`** (append — הטאב מציג תמיד את **האחרון במערך** כ"היום") כאובייקט חדש:
   ```json
   {
     "id": "gk-<YYYYMMDD>",
     "icon": "<אמוג'י מתאים לנושא>",
     "title": "<כותרת קצרה ומעניינת>",
     "tip": "<משפט תקציר אחד — 'מה תלמד היום'>",
     "md": "<גוף הנושא ב-Markdown: כותרות ##, פסקאות, ושורת '> 🤔 האם ידעת?' בסוף>"
   }
   ```
   - ה-`id` חייב להיות ייחודי (לפי תאריך). אם כבר קיים נושא להיום — אל תכפיל, החלף אותו (במקום, לא בסוף).
   - **אל תוסיף שדה `level`** — אין רמות בקובץ הזה, רק רשימה שגדלה.
   - שמור את כל הנושאים הקיימים. ה-`md` הוא מחרוזת אחת (escape לתווי שורה כ-`\n`).
   - ודא שה-JSON נשאר תקין.
3. **עדכן את `app/data/general-knowledge-topics.json`**: הוסף את נושא היום למערך `topics` (שם קצר, למשל "נפילת האימפריה הרומית"). אל תמחק נושאים קיימים. ודא JSON תקין.
4. **בצע commit ו-push** של שני הקבצים: `Add general knowledge topic <YYYY-MM-DD>`.
5. ⚠️ **חובה — הסביבה שלך עובדת על branch נפרד, לא על `main`.** מיד אחרי ה-push, צור Pull Request אל `main` באמצעות `mcp__github__create_pull_request` ומזג אותו מיד עם `mcp__github__merge_pull_request` (`merge_method: "squash"`), **באותה הרצה**. אם יש merge conflicts: `git fetch origin main && git merge -X ours origin/main --no-edit && git push`, ואז מזג שוב. ודא שהתגובה מכילה `"merged": true` לפני שאתה מסיים — בלי זה הנושא לא יגיע לאפליקציה.

> כך כל בוקר נוסף נושא ידע כללי חדש לטאב הלימוד, הנושא של היום מקבל תג "היום", והנושאים הישנים נשמרים לגלילה.

```
## <פרטים והקשר>
<הרחבה עם עובדות מדויקות ומאומתות>

```
