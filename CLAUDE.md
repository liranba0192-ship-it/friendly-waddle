# CLAUDE.md — חלבונינץ 🌅

מסמך זה מגדיר לקלוד קוד את הפרויקט, ארכיטקטורת האוטומציה, וכללי הפעולה. קרא אותו **לפני כל הרצה**.

---

## מה הפרויקט הזה

**חלבונינץ** היא אפליקציית כושר אישית (PWA לאייפון) שמשלבת:
- **בריפינג ידע יומי אוטומטי** — תוספים, אימון, תזונה (בעברית, ~7 דקות קריאה)
- **מעקב אימונים** — Progressive Overload
- **יומן אוכל ושקילה**
- **מודול לימוד** — אנגלית לבגרות, פיננסים, AI

האפליקציה מתארחת בגיטהאב Pages מהענף `main` ומתעדכנת אוטומטית בכל push.

---

## ארכיטקטורת האוטומציה — 4 Routines

קלוד קוד מריץ 4 שגרות מתוזמנות. כל שגרה קוראת קובץ prompt, מייצרת תוכן, ודוחפת לריפו.

| Routine | Prompt file | פלט | תדירות |
|---------|-------------|-----|---------|
| בריפינג בוקר | `daily-briefing-prompt.md` | `briefings/YYYY-MM-DD.md` | יומי |
| קריאה באנגלית | `daily-reading-prompt.md` | `readings/YYYY-MM-DD.md` | יומי |
| שיעור פיננסי | `daily-finance-prompt.md` | רשומה ב-`app/data/finance.json` | יומי |
| שיעור AI | `ai-guide-prompt.md` | רשומה ב-`app/data/ai-guide.json` | יומי/שבועי |

---

## מפת קבצים קריטיים

```
briefings/
  covered-topics.json   ← "פנקס" כל הנושאים שכבר נכתבו — חובה לקרוא לפני בחירת נושא
  index.json            ← אינדקס כל הבריפינגים — חובה לעדכן בכל הרצה
  YYYY-MM-DD.md         ← הבריפינגים עצמם

readings/
  index.json            ← אינדקס כל הקריאות
  YYYY-MM-DD.md         ← קריאות הבוקר

app/data/
  finance.json          ← מסלול שיעורי הפיננסים (גדל מדי יום)
  ai-guide.json         ← מדריך ה-AI (גדל מדי יום/שבוע)
  foods.json            ← מאגר מזון (קבוע)
  exercises.json        ← מאגר תרגילים (קבוע)
  vocab.json            ← 886 מילים לבגרות (קבוע)

app/                    ← PWA — אל תשנה ללא צורך
```

## כללי ברזל (חלים על כל Routine)

### אנטי-כפילויות
- **לפני בחירת נושא לבריפינג:** קרא `briefings/covered-topics.json` במלואו. אסור לחזור על נושא שכבר מופיע שם, כולל וריאציות קרובות.
- **לפני בחירת שיעור פיננסי/AI:** קרא את ה-JSON המתאים ועבור על כל ה-`id`/`title` הקיימים.

### אמינות מידע
- אל תמציא מחקרים, מספרים, מינונים, שמות חוקרים, או URL-ים.
- כל נתון מספרי חייב לבוא ממקור שמצאת ב-WebSearch בפועל.
- כשהראיות חלשות — אמור זאת במפורש. עדיף קצר ומדויק מארוך ומנופח.

### שמירה ו-git (חובה בסוף כל הרצה)
1. שמור את הקובץ/קבצים החדשים.
2. עדכן את ה-`index.json` הרלוונטי.
3. עדכן `briefings/covered-topics.json` (בשגרת הבריפינג בלבד).
4. בצע `git add` לכל הקבצים שנשתנו.
5. בצע `git commit` עם הודעה תיאורית (למשל `Add daily briefing 2026-06-28`).
6. בצע `git push origin main`.

## זרימת עבודה — שגרת הבריפינג (הכי מורכבת)

1. קרא `briefings/covered-topics.json` — זו רשימת הנושאים האסורים.
2. בחר 3 נושאים חדשים (תוסף / עקרון אימון / תזונה) שאינם ברשימה.
3. WebSearch לכל נושא עם התאריך הנוכחי.
4. כתוב את הבריפינג לפי הפורמט ב-`daily-briefing-prompt.md`.
5. שמור ל-`briefings/YYYY-MM-DD.md`.
6. עדכן `briefings/index.json` — הוסף רשומה בראש המערך.
7. עדכן `briefings/covered-topics.json` — הוסף את 3 הנושאים לרשימות המתאימות.
8. Commit ו-push של 3 הקבצים ל-`main`.

## מה לא לגעת בו

- `app/` — קוד ה-PWA (HTML/CSS/JS). שנה רק אם התבקשת במפורש.
- `app/data/foods.json`, `vocab.json`, `exercises.json` — נתונים קבועים.
- `SETUP.md` — מדריך הגדרה לאנושי.
- `index.html` בשורש.
