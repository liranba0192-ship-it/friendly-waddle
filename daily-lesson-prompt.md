# השיעור היומי — כושר, תזונה ובריאות 📅💪

> זהו ה-prompt שטריגר מתוזמן (Scheduled Trigger) של Claude Code מריץ כל בוקר.
> המטרה: **שיעור בוקר אחד קצר, מבוסס-מחקר, חדש לגמרי** — שמופיע בטאב "לימוד → שיעור יומי" באפליקציה.
> אין צורך לערוך אותו ידנית — פשוט חבר אליו את הטריגר (ראה `SETUP.md`).

## המשימה

הפק **שיעור לימוד יומי אחד בעברית** בנושא כושר / תזונה / תוספים / בריאות — קצר, ברור ומבוסס-ראיות.
היקף: קריאה של **2–3 דקות** (בערך 250–450 מילים). מוסבר מאפס, בלי ז'רגון, עם דירוג ראיות.

## 🔄 אסור לחזור על נושא — מנגנון פנקס כפול (חובה מוחלטת — קרא ובצע קודם!)
**הבעיה שאנו פותרים: אסור לחזור על נושא שכבר נכתב — לא בשיעור היומי, וגם לא בתדריכי הבוקר.**

יש שני פנקסים שצריך לקרוא **לפני** בחירת הנושא:
1. **`app/data/daily-lessons-topics.json`** — כל הנושאים שכבר נכתבו בשיעור היומי.
2. **`briefings/covered-topics.json`** — כל הנושאים שכבר נכתבו בתדריכי הבוקר (תוספים/אימון/תזונה).

בצע בדיוק כך, לפי הסדר:
1. **קרא את שני הקבצים** במלואם. כל מה שמופיע בהם — אסור.
2. בחר **נושא חדש לגמרי** שאינו מופיע באף אחד מהם — לא הנושא עצמו ולא וריאציה קרובה (למשל אם "אומגה-3" קיים — אסור גם "שמן דגים"; אם "קפאין" קיים — אסור "קפאין לאימון").
3. הנושא חייב להיות בתוך התחומים: **כושר / תזונה / תוספים / פיזיולוגיה ובריאות**.
4. בסס על **מחקר עדכני וטרי**: WebSearch עם **התאריך הנוכחי** ו-"latest / 2026 / meta-analysis".

## ⛔ כללי-ברזל נגד מידע לא מבוסס
1. **אסור להמציא** מחקרים, מספרים, מינונים או ציטוטים. אם לא מצאת מקור אמין — אל תכתוב את הטענה.
2. כל מינון/אחוז חייב לבוא ממקור שמצאת ב-WebSearch, לא מהזיכרון.
3. כשהראיות חלשות או חסרות — אמור זאת במפורש.
4. דרג תמיד את חוזק הראיות: 🟢 חזק / 🟡 בינוני / 🔴 מוקדם-מוגבל.

## 🗣️ סגנון — הסבר כמו למתחיל מוחלט
1. הנח שלקורא אין ידע מוקדם. הסבר כל מושג מאפס.
2. פרק כל מונח מקצועי במילים פשוטות לפני שמשתמשים בו.
3. שפה ברורה וידידותית, עם דוגמאות מהחיים.
4. כלול **טיפ פרקטי אחד** ליישום מיידי.

## 💾 שמירה לאפליקציה (חובה — בסוף ההרצה)

1. **קרא את `app/data/daily-lessons.json`.** מבנה: `{ "note": ..., "lessons": [ ... ] }`.
2. **הוסף את שיעור היום בסוף מערך `lessons`** (append — כדי לא לשבש את סדר הרוטציה לפי תאריך) כאובייקט חדש:
   ```json
   {
     "id": "dl-<YYYYMMDD>",
     "icon": "<אמוג'י מתאים>",
     "title": "<כותרת קצרה וברורה>",
     "tip": "<משפט תקציר אחד>",
     "level": "🆕 שיעורי המשך",
     "md": "<גוף השיעור ב-Markdown: כותרות ##, פסקאות, ושורת טיפ '> 💡 ...', ושורת 'רמת ראיות:' בסוף>"
   }
   ```
   - ה-`id` חייב להיות ייחודי (לפי תאריך). אם כבר קיים שיעור להיום — אל תכפיל, החלף אותו.
   - שמור את כל השיעורים הקיימים. ה-`md` הוא מחרוזת אחת (escape לתווי שורה כ-`\n`).
   - ודא שה-JSON נשאר תקין.

> ℹ️ הערה: השיעור היומי באפליקציה כבר עובד **בלי רוטיין** — יש 100 שיעורים קבועים שמתחלפים לבד לפי תאריך. שגרה זו היא **רשות בלבד** — היא רק מוסיפה עוד שיעורים מעבר ל-100, אם תרצה תוכן שמתעדכן עם מחקרים חדשים.
3. **עדכן את `app/data/daily-lessons-topics.json`**: הוסף את נושא היום למערך `topics` (שם גנרי, למשל "ויטמין C"). אל תמחק נושאים קיימים. ודא JSON תקין.
4. **בצע commit ו-push** של שני הקבצים ל-branch `main`:
   `Add daily lesson <YYYY-MM-DD>`.

> כך כל בוקר נוסף שיעור חדש לטאב הלימוד, השיעור של היום מקבל תג "היום", והשיעורים הישנים נשמרים לגלילה.
