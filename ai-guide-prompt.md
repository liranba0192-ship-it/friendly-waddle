# שיעור AI חדש לפי החדשות בתחום 🤖🚀

> זהו ה-prompt שטריגר מתוזמן (Scheduled Trigger) של Claude Code מריץ **כל יום/שבוע**.
> המטרה: שמדריך ה-AI באפליקציה **יתעדכן עם העתיד** — שיעור חדש על מה שהכי עדכני ומוביל בתחום ה-AI כרגע.
> חבר אליו טריגר (ראה `SETUP.md`).

## המשימה
הוסף **שיעור AI אחד חדש** לקובץ `app/data/ai-guide.json` — קצר, ברור, בעברית, מעשי, ומבוסס על **התפתחות עדכנית אמיתית** בתחום ה-AI.

## 🔄 נושא חדש ועדכני (חובה — קרא קודם!)
1. קרא את `app/data/ai-guide.json` ועבור על **כל השיעורים הקיימים** (שדות `id` ו-`title`) כדי לא לחזור.
2. בצע **WebSearch עם התאריך הנוכחי** ("latest AI", "new AI models 2026", "AI agents news" וכו') כדי למצוא התפתחות, יכולת, מודל, כלי או טכניקה **חדשים ורלוונטיים** שעדיין לא מכוסים במדריך.
3. בחר נושא שמעשי ללומד — איך זה עוזר לו לעבוד טוב יותר עם AI — לא רק "חדשות".

## כללי איכות (קריטי)
1. **אל תמציא** יכולות, שמות מוצרים, מחירים, מספרים או מחקרים. כל טענה עובדתית/מספרית חייבת להגיע ממקור אמיתי שמצאת ב-WebSearch.
2. עברית פשוטה מהיסוד; הסבר כל מונח חדש (סוכן, מודל, הקשר וכו').
3. כלול **דוגמה מעשית** אחת לפחות (רצוי פרומפט ❌ חלש מול ✅ חזק).
4. אורך כמו השיעורים הקיימים (~300–430 מילים Markdown), ומסתיים בשורת `> 💡`.
5. הדגש שהאחריות לבדוק את הפלט נשארת על המשתמש.

## פורמט — הוסף אובייקט אחד למערך `lessons`
הוסף **בסוף** מערך `lessons` (רמת "להמשך החיים") אובייקט במבנה:
```json
{
  "id": "<מזהה-אנגלי-קצר-ייחודי>",
  "icon": "<אמוג'י>",
  "title": "<כותרת קצרה בעברית>",
  "level": "♾️ להמשך החיים",
  "md": "## <כותרת>\\n<תוכן Markdown עם \\n לשורות, דוגמה, ו-> לטיפ>",
  "tip": "<משפט טיפ מעשי>"
}
```
- שדה `md` הוא מחרוזת אחת — השתמש ב-`\\n` למעברי שורה.
- ודא **id ייחודי** שלא קיים, ושדה `level` בדיוק `"♾️ להמשך החיים"` כדי שייכנס לרמה הנכונה.
- ודא JSON **תקין לחלוטין** אחרי ההוספה.

## 💾 שמירה (חובה — בסוף ההרצה)
1. כתוב את `app/data/ai-guide.json` המעודכן.
2. ודא תקינות JSON (`python3 -c "import json;json.load(open('app/data/ai-guide.json'))"`).
3. בצע **commit ו-push**. הודעת commit: `Add AI lesson: <title>`.
4. ⚠️ **חובה — הסביבה שלך עובדת על branch נפרד, לא על `main`.** מיד אחרי ה-push, צור Pull Request אל `main` באמצעות `mcp__github__create_pull_request` ומזג אותו מיד עם `mcp__github__merge_pull_request` (`merge_method: "squash"`), **באותה הרצה**. אם יש merge conflicts: `git fetch origin main && git merge -X ours origin/main --no-edit && git push`, ואז מזג שוב. ודא שהתגובה מכילה `"merged": true` לפני שאתה מסיים.

> כך מדריך ה-AI ממשיך לגדול ולהתעדכן עם ההתפתחויות — הלומד תמיד נשאר ברמת החזית.
