# תכנון התקנות מיזוג — AC Installation Planner

אפליקציית ווב (PWA) לסימון מסלולי צנרת על תוכנית/צילום וחישוב אורכים אמיתיים
אוטומטית. עברית RTL מלאה, כפתורים גדולים, שמירה אוטומטית. מיועדת לאנשי מיזוג.

> זו **שלב 1 (MVP)**. שלבים 2–3 (PDF, חדרים/יחידות, מחשבון BTU, גדלי צנרת,
> ייצוא ושיתוף PDF) מתוכננים בהמשך.

## מה עובד בשלב 1
- **רשימת פרויקטים** לפי לקוח — יצירה / פתיחה / שינוי שם / מחיקה + חיפוש.
- **העלאת תוכנית** (צילום JPG/PNG) כרקע לקנבס.
- **כיול קנה מידה** באשף 3-לחיצות: נקודה A → נקודה B → הקלדת המרחק האמיתי.
- **ציור 3 סוגי מסלולים** עם העט (נקודה-נקודה): 🟦 גז · 🟩 ניקוז · 🟧 חשמל.
  אפשר כמה מסלולים נפרדים לכל סוג.
- **חישוב אורכים** אמיתי במטרים, מתעדכן חי (`סיכום אורכים`).
- **עריכה מלאה**: גרירת קודקודים, הוספת/מחיקת נקודה, מחיקת קו, **Undo/Redo**.
- **שמירה אוטומטית** (אופליין-פירסט ל-IndexedDB), והתקנה כ-PWA לאייפד.

## הרצה מקומית
```bash
npm install
npm run dev      # פיתוח
npm run build    # בנייה לפרודקשן (dist/)
npm run preview  # תצוגה מקדימה של ה-build
npm test         # בדיקות יחידה למתמטיקה (geometry/transform/scale)
```
ללא הגדרת Supabase (ראו למטה) — האפליקציה רצה **במצב מקומי בלבד**: בלי מסך
התחברות, כל הנתונים ב-IndexedDB של הדפדפן. מצוין לבדיקה מיידית.

## הפיכה למרובת-משתמשים (ענן) — אופציונלי
כדי שכל מתקין יקבל חשבון משלו עם סנכרון בין מכשירים:
1. צור פרויקט חינמי ב-[Supabase](https://supabase.com).
2. הרץ את `supabase/schema.sql` ב-SQL Editor (יוצר טבלה, RLS, ו-bucket תמונות).
3. העתק `.env.example` ל-`.env` ומלא:
   ```
   VITE_SUPABASE_URL=...
   VITE_SUPABASE_ANON_KEY=...
   ```
4. הרץ מחדש. עכשיו יש מסך התחברות (אימייל+סיסמה), וכל משתמש רואה רק את שלו
   (נאכף ב-RLS).

> מפתח ה-anon בטוח בצד-לקוח **רק** בזכות RLS. אל תשבית RLS.

## פריסה ל-GitHub Pages
1. העבר את תוכן `ac-planner/` לריפו משלו (התיקייה עצמאית לחלוטין).
2. ב-Settings → Pages: Source = GitHub Actions.
3. הוסף Secrets (אם משתמשים בענן): `VITE_SUPABASE_URL`, `VITE_SUPABASE_ANON_KEY`.
4. עדכן את `base` ב-`vite.config.js` לשם הריפו (כרגע `/ac-planner/`).
5. כל push ל-`main` מפרסם אוטומטית (`.github/workflows/deploy.yml`).

## ארכיטקטורה (למתחזק)
- **React + Vite + react-konva** (JS רגיל). הקנבס ב-`src/editor/`.
- **כל המתמטיקה ב-`src/lib/`** כפונקציות טהורות (geometry / transform / scale)
  — מבודדות וניתנות לבדיקה. זה החלק הרגיש; בדוק שם קודם.
- **נקודות נשמרות ב-image-space** (פיקסלים יחסית לתמונה המקורית), לא ב-screen.
  לכן האורכים נכונים בכל זום/הזזה, וכיול מחדש מעדכן הכל "בחינם".
- **Undo/Redo**: `src/hooks/useHistory.js` (snapshots). גרירת קודקוד = צעד אחד.
- **נתונים**: Dexie (IndexedDB) מקור-אמת מקומי; דחיפה/משיכה אופציונלית ל-Supabase
  (`src/db/sync.js`), last-write-wins לפי `updated_at`.

מבנה התיקיות והחלטות התכנון המלאות — בתוך הקוד (תגובות בעברית).
